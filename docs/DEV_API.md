# Development API и `soprctl`

Development API предназначен для интерактивной диагностики запущенного
приложения и автоматизации воспроизводимых сценариев. По умолчанию он доступен
только на loopback-интерфейсе и требует токен из discovery-файла:

```bash
sopr-scheme-gener --dev-api --dev-port 0
soprctl info
```

Порт `0` просит ОС выбрать свободный порт. `soprctl` автоматически читает
адрес, порт и токен из discovery-файла. Для отдельного экземпляра можно
совместно задать `--dev-info-file` приложению и `--info-file` клиенту.

## Структурный документ

`soprctl document` возвращает JSON-снимок активной задачи со стабильными
корневыми разделами:

| Путь | Назначение | Запись |
| --- | --- | --- |
| `/task_type` | стабильный id, заголовок и индекс типа задачи | нет |
| `/canvas` | размеры холста | `/canvas/width`, `/canvas/height` |
| `/common` | общие параметры по сериализованным меткам | один параметр |
| `/settings` | параметры типа задачи по сериализованным меткам | один параметр |
| `/text` | текст задачи, если редактор предусмотрен | целиком |
| `/task` | JSON-представление модели задачи | существующие поля и элементы списков |

Пути используют JSON Pointer (RFC 6901). Например:

```bash
soprctl doc-get /task/sections/0/l
soprctl doc-set /task/sections/0/l 2.5
soprctl doc-set /task/sections/0/qx '"+"'
```

Значение `doc-set` разбирается как JSON, а если это не JSON — передаётся как
обычная строка. Закрытые атрибуты, замена корня модели и создание новых полей
запрещены.

Несколько изменений выполняются одним атомарным patch: при ошибке исходные
значения восстанавливаются, после успеха перерисовка происходит один раз.

```bash
soprctl patch --json '[
  {"path": "/task/sections/0/l", "value": 3.0},
  {"path": "/canvas/width", "value": 640}
]'
```

## Сценарий одним запросом

Команда может выбрать тип задачи, применить patch, дать Qt обработать таймеры
и события, снять PNG и вернуть итоговый документ:

```bash
soprctl scenario \
  --json '{
    "selector": "beams",
    "changes": [{"path": "/task/sections/0/l", "value": 2.5}],
    "wait_ms": 50,
    "screenshot_target": "canvas"
  }' \
  --screenshot-output /tmp/beams-2.5.png
```

`wait_ms` ограничен диапазоном 0–10000. `screenshot_target` принимает те же
имена, что `soprctl screenshot`: `window`, `central`, `canvas_container`,
`canvas` или `settings`. Чтобы не возвращать полный снимок, передайте
`"include_document": false`.

## Диагностика

Журнал событий содержит монотонный `sequence`, время, имя события и данные.
Курсор позволяет запрашивать только новые записи:

```bash
soprctl events
soprctl events --since 12 --limit 50
soprctl events --clear
```

Ошибки протокола и рендера доступны отдельно. При включённом Development API
ошибка рендера записывается в журнал вместо показа блокирующего диалога:

```bash
soprctl errors
soprctl errors --since 3
soprctl errors --clear
```

Низкоуровневый вызов любого метода доступен через `soprctl raw METHOD
--params JSON`.

## Небезопасный Python

`soprctl exec` выключен по умолчанию и включается только явным флагом
`--unsafe-dev-exec`. Он даёт коду полный доступ к процессу приложения и не
является частью безопасного структурного API.
